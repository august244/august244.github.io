任意用户类漏洞

如果在某一个功能点存在下面的漏洞，可能其他功能点都存在这个漏洞

四（六）位验证码可爆破

出现的位置

  1. 只要是发短信的都可能存在

挖掘方法

  1. 尝试底频率爆破

验证码回显

挖掘方法

  1. 就直接看返回包

只验证了验证码，没有验证绑定

简单的说就是拿别人的验证码，自己使用

修改返回包

bp 修改返回的状态

可以将返回包改为

false -> success true

400 -> 200 0 1 -1

发送验证码双写

比如

mobile = a,b,c

或者是

mobile =a&mobile =b

有些时候甚至可以在接受到code

然后携带着code 进行请求

mobile=a,b,c&code=6722..

第三方登录

拦截返回包

比如使用qq=111 进行认证，截取返回包然后修改为qq=222 进行认证

随意的验证码

就是随便一个验证码都能能够实现功能

验证码为null 或者不写

短信发送时绕过

我们本身接受验证码之后，重置别人的 账号

用户名枚举

如果出现了使用用户名登录的情况下，可以尝试验证用户名枚举

手机号泄露

尝试一个用户，他响应包中可能会出现手机号

重置链接可预测
