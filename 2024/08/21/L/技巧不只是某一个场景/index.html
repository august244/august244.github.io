技巧不只是某一个场景

痛点

  1. 不要执着于每一个src，或者某一个系统
  2. 挖掘漏洞需要时间的积累
  3. 多学习，多挖洞
  4. 先练习好某一个类型的漏洞，再去多尝试其他的漏洞

双写不仅仅是手机号

双写的方式有两种

  1. 双写参数

  2. 逗号

    如果userid - userid2

垃圾字符

很多时候垃圾字符都是顺手一加

+86

x

. 注意这个是. 是每一次增加一个

,1(这个值是遍历) 比如user,1 user,2 …….

删除多余的字符

一带一

也就是说，这种情况下必须，必须先去实现网站的基本功能，你就会发现很多剩下的功能了

删除多余的数据

无中生有

也就是添加参数

万能数据

  1. 万能的验证码

    比如111111等等

  2. 万能的手机号

    11111111111等等

  3. 只要是填写数据或者是认证的地方就可以尝试

    比如邀请码等等

修改状态

0 -> 1

false -> true

500 -> 200

路径穿越（bypass 403）

  1. //

  2. /../

  3. 比如在ssrf 的时候

    当我们使用http://127.0.0.1 发现存在问题

  4. /..;/

大小写

就是可以尝试将大写的变成小写的

或者是小变大

这种情况是由于网站使用的是拦截器，但是没有对参数完整的过滤，只拦截对应的参数

敏感参数的抓取

比如某一个点需要进行判断你是否达到要求

然后这种往往都是一个一个参数进行控制

所以尝试删除对应的参数，或者将false -> true

场景一

场景二

垂直越权，赋予了子用户的权限，然后收回，发现短时间还能使用这个功能的话，那就是存在漏洞
